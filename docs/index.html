The Problem

More skills,
worse results.

01.

Context Window Bloat

Every installed skill's description loads into context on every turn — whether it's relevant or not. With 30+ skills, this consumes thousands of tokens and dilutes the model's attention across the board.

02.

Routing Conflicts

Similar skill descriptions cause the agent to invoke the wrong skill. An animation skill fires when you need a polish skill. A Python skill triggers on Go code. Users just notice their agent "feels dumber."

03.

No Management Layer

There's no npm ls for skills — no unified view of what's installed, what's stale, or what's overlapping. When skills conflict, there's no tool to diagnose why.

How It Works

N skills → 1 router.
Originals stay put.

Before — 6 descriptions in context
.claude/skills/
  animate/
  polish/
  critique/
  adapt/
  bolder/
  colorize/
  ↑ all 6 descriptions loaded
    on every agent turn
After — 1 description in context
.claude/skills/
  animate/       ← untouched
  polish/        ← untouched
  critique/      ← untouched
  adapt/         ← untouched
  bolder/        ← untouched
  colorize/      ← untouched
  ui-design/     ← router
    SKILL.md     ← 1 description
    _manifest.json

The router's SKILL.md contains a synthesised description covering all sub-skills, a dispatch table pointing to each original, and a capabilities section. When triggered, the agent reads only the one relevant original — not all of them.

Workflow

Six steps.
Done.

1

Detect

Auto-detects which AI agent tool directories exist in the current project.

2

Determine

Identifies which skills to group — user-specified or auto-suggested by collection.

3

Synthesize

Reads sub-skill descriptions and writes a unified router description (WHAT + WHEN + NOT-WHEN).

4

Preview

Dry-run generates the full SKILL.md for review before any files are written.

5

Create

Writes the router skill directory with SKILL.md and _manifest.json. Originals untouched.

6

Explain

Tells you how to invoke the router (/router-name) and sub-skills directly (/skill-name).

Get Started

Install in
one copy.

# Claude Code (local project)

# Any universal agent
cp -r skills/skill-router/ .agents/skills/skill-router/

# Then tell your agent:
/skill-router
